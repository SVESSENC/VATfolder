# VATfolder

## Project Structure
- `docs/architecture/`: architecture decisions, system outline, and tech stack
- `docs/diagrams/`: Mermaid diagrams split by concern
- `docs/roles/`: role instructions (`architect`, `developer`, `designer`, `researcher`, `reviewer`)
- `docs/research/`: research notes and legal discovery artifacts
- `api/`: API contract (`openapi.yaml`)
- `database/`: schema and migration SQL
- `team.mcp.yaml`: team workflow and orchestration config

## Start Here
1. Read `docs/architecture/tech-stack.md`
2. Read `docs/architecture/architecture.md`
3. Review `docs/diagrams/README.md`
4. Implement against `api/openapi.yaml` and `database/db_schema.sql`
